# RAG Query Logging Contract

**Purpose**: Track RAG system performance and user queries for analytics  
**Consumer**: Phi's Dashboard, analytics pipeline  
**Database**: `rag_query_logs` table (managed by Phi)

## Log Entry Schema

Each query to the RAG system should be logged with this structure:

```json
{
  "id": "uuid",
  "question": "What is the refund policy?",
  "document_id": "doc_001",
  "retrieved_chunk_ids": [
    "doc_001_page_1_chunk_000",
    "doc_001_page_1_chunk_001",
    "doc_001_page_2_chunk_000"
  ],
  "retrieval_scores": [0.87, 0.82, 0.75],
  "num_chunks_retrieved": 3,
  "generated_response": "Refunds are available within 30 days...",
  "answer_confidence": 0.85,
  "retrieval_latency_ms": 45,
  "llm_latency_ms": 1200,
  "total_latency_ms": 1245,
  "model_name": "all-MiniLM-L6-v2",
  "llm_model_name": "gpt-3.5-turbo",
  "status": "success",
  "user_id": "user_123",
  "session_id": "session_456",
  "created_at": "2024-01-15T10:30:45.123Z",
  "metadata": {
    "source": "chatbot_ui",
    "query_language": "en",
    "filter_applied": {"document_id": "doc_001"}
  }
}
```

## Field Definitions

### Identifiers

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | UUID | Yes | Unique log entry identifier |
| `question` | string | Yes | The user's query |
| `user_id` | string | No | User who asked the question |
| `session_id` | string | No | Chat session identifier |

### Retrieval

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `document_id` | string | No | Filtered document ID (if applicable) |
| `retrieved_chunk_ids` | array[string] | Yes | List of chunk IDs returned |
| `retrieval_scores` | array[number] | Yes | Similarity scores for each chunk (0.0-1.0) |
| `num_chunks_retrieved` | integer | Yes | Count of chunks (should equal length of arrays above) |

### Answer

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `generated_response` | string | No | The answer generated by LLM (null if retrieval-only) |
| `answer_confidence` | number (0-1) | No | Confidence score for generated answer |

### Performance

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `retrieval_latency_ms` | integer | Yes | Time to search vector store |
| `llm_latency_ms` | integer | No | Time to generate answer (if applicable) |
| `total_latency_ms` | integer | Yes | Total request time end-to-end |

### Models

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `model_name` | string | Yes | Embedding model (e.g., "all-MiniLM-L6-v2") |
| `llm_model_name` | string | No | LLM used for answer (e.g., "gpt-3.5-turbo") |

### Status

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `status` | enum | Yes | One of: `success`, `partial_success`, `error`, `no_results` |

### Metadata

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `source` | string | No | Where query came from (e.g., "chatbot_ui", "api", "dashboard") |
| `query_language` | string | No | Language of the question (e.g., "en", "es") |
| `filter_applied` | object | No | Any filters applied to the query |
| `created_at` | ISO8601 | Yes | Timestamp of the query |

## SQL Schema

```sql
CREATE TABLE rag_query_logs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    
    -- Query Info
    question TEXT NOT NULL,
    user_id VARCHAR(255),
    session_id VARCHAR(255),
    
    -- Retrieval
    document_id VARCHAR(255),
    retrieved_chunk_ids TEXT[] NOT NULL,
    retrieval_scores NUMERIC[] NOT NULL,
    num_chunks_retrieved INTEGER NOT NULL,
    
    -- Answer
    generated_response TEXT,
    answer_confidence NUMERIC,
    
    -- Performance
    retrieval_latency_ms INTEGER NOT NULL,
    llm_latency_ms INTEGER,
    total_latency_ms INTEGER NOT NULL,
    
    -- Models
    model_name VARCHAR(255) NOT NULL,
    llm_model_name VARCHAR(255),
    
    -- Status
    status VARCHAR(50) NOT NULL,
    
    -- Metadata
    source VARCHAR(100),
    query_language VARCHAR(10),
    filter_applied JSONB,
    
    -- Timestamps
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Indexes for analytics queries
CREATE INDEX idx_rag_logs_user_id ON rag_query_logs(user_id);
CREATE INDEX idx_rag_logs_session_id ON rag_query_logs(session_id);
CREATE INDEX idx_rag_logs_document_id ON rag_query_logs(document_id);
CREATE INDEX idx_rag_logs_status ON rag_query_logs(status);
CREATE INDEX idx_rag_logs_created_at ON rag_query_logs(created_at);
CREATE INDEX idx_rag_logs_source ON rag_query_logs(source);
```

## Logging Integration Points

### In RAG Response

Lap should include logging context when returning results:

```python
from rag.rag_pipeline import RAGPipeline

pipeline = RAGPipeline()
result = pipeline.query("What is the policy?")

# Result includes:
{
    "question": "What is the policy?",
    "retrieved_context": [...],
    "metadata": {
        "retrieval_time_ms": 45,
        "num_chunks_searched": 2500,
        ...
    }
}

# Phi's backend logs:
log_entry = {
    "question": result["question"],
    "retrieved_chunk_ids": [c["chunk_id"] for c in result["retrieved_context"]],
    "retrieval_scores": [c["score"] for c in result["retrieved_context"]],
    "retrieval_latency_ms": result["metadata"]["retrieval_time_ms"],
    "model_name": result["model"],
    ...
}
```

## Analytics Queries

### Query Volume

```sql
SELECT 
    DATE(created_at) as date,
    COUNT(*) as query_count,
    COUNT(DISTINCT user_id) as unique_users
FROM rag_query_logs
WHERE created_at >= NOW() - INTERVAL '7 days'
GROUP BY DATE(created_at)
ORDER BY date;
```

### Average Retrieval Performance

```sql
SELECT 
    AVG(retrieval_latency_ms) as avg_retrieval_ms,
    MAX(retrieval_latency_ms) as max_retrieval_ms,
    PERCENTILE_CONT(0.95) WITHIN GROUP (ORDER BY retrieval_latency_ms) as p95_retrieval_ms
FROM rag_query_logs
WHERE status = 'success';
```

### Answer Confidence Distribution

```sql
SELECT 
    ROUND(answer_confidence::numeric, 1) as confidence_bin,
    COUNT(*) as count
FROM rag_query_logs
WHERE generated_response IS NOT NULL
GROUP BY ROUND(answer_confidence::numeric, 1)
ORDER BY confidence_bin DESC;
```

### Most Retrieved Chunks

```sql
SELECT 
    chunk_id,
    COUNT(*) as frequency,
    AVG(r.score) as avg_score
FROM rag_query_logs
CROSS JOIN LATERAL UNNEST(retrieved_chunk_ids) as chunk_id
WHERE created_at >= NOW() - INTERVAL '30 days'
GROUP BY chunk_id
ORDER BY frequency DESC
LIMIT 10;
```

### Success Rate by Source

```sql
SELECT 
    source,
    COUNT(*) as total_queries,
    COUNT(*) FILTER (WHERE status = 'success') as successful,
    ROUND(100.0 * COUNT(*) FILTER (WHERE status = 'success') / COUNT(*), 2) as success_rate
FROM rag_query_logs
GROUP BY source;
```

## Data Retention

- **Keep indefinitely**: Question text, answers, metadata
- **Archive after 90 days**: Detailed performance metrics
- **Delete after 1 year**: Session identifiers, user IDs (for privacy)

## Privacy Considerations

- Do NOT log sensitive user information beyond user_id
- Do NOT store full context in logs (only chunk IDs)
- Anonymous mode: Omit user_id and session_id for privacy-conscious queries
- GDPR: Support "right to be forgotten" by purging user's logs
